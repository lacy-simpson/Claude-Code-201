Claude Code Overview

Claude shows up in a few different places, and each one fits a different kind of work. Before we open the app, it helps to see where Claude Code sits in that lineup and what makes it the most capable of the group.

Lesson Objective

Place Claude Code among the other Claude surfaces, describe what it can do that the others cannot, and explain what an Agent Skill is.

The Claude Ecosystem

The same Claude is available through several interfaces. They get progressively more capable, and more powerful, as you move from a steered conversation toward a tool that can act on your machine.

This course focuses on Claude Code, the most capable surface. Browse the others so you know when something lighter is the better choice.

Claude Chat
Conversation
What it is

A conversation you steer turn by turn. It can reason, draft, and generate visualizations, but it does not touch your files or systems.

Where it lives

Claude Plugins
Inside your apps
What it is

Claude available inside approved work tools, using the surrounding app context you allow: a thread, a spreadsheet, a document, or a deck.

Use when
You want Claude's help without leaving the app where the work already lives, and you review changes before they are saved or sent.
Claude Cowork
Desktop app
What it is

A delegated-work mode that can read and write files on your machine, use connected apps, and run skills. Stronger than Chat, with live artifacts for visualization.

Use when
You want to hand off a bounded job across your files and review the result.

What is Claude Code?

Claude Code is an agentic AI assistant that reads your files, runs commands, and edits work on your behalf, with your permission. It acts only where you point it, and when your permissions are set up well it waits for your approval before doing anything consequential.

It's accessible

Use it in the desktop app on the Code tab, in the terminal, or in your editor. The desktop app is the easiest place to start.

It's interactive

It is an agent, not just an advisor. It plans multi-step work and is shaped by the permissions and context you give it.

It's extensible

Skills, connectors, and command-line tools teach it a workflow once so you can reuse it across sessions.

What is an Agent Skill?

A skill is a reusable playbook you give Claude for a specific task. Instead of explaining what you need from scratch every session, a skill captures it once: the steps, the rules, and the expected output. Claude picks it up automatically when your request matches.

A skill is mostly just a folder with a text file. The most important file is SKILL.md. Its top section, called frontmatter, declares the skill's name and description. The rest is plain Markdown: the workflow, the rules, and examples.

A skill on disk 
meeting-notes-summarizer/
  SKILL.md          # the instructions Claude reads
  scripts/          # optional: code the skill can run for repeatable steps
    format.py
  references/       # optional: extra docs Claude loads only when needed
Skills and scripts together

Skills handle judgment, context, and variation: the things that change case by case. Scripts handle what is fast and fixed: retrieval, formatting, and predictable transformations. Together they are more reliable than either alone, because Claude does not have to reinvent the routine steps every time.
