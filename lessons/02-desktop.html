Know your workspace

The Claude Code Desktop Interface

The Claude Code window has a handful of controls, and once you can name each one the app stops feeling mysterious. This is a guided tour. We point at every control, say what it does, and note which lesson goes deeper.

Lesson Objective

Find and name the working-folder picker, permission-mode selector, attachments, model and effort controls, the usage meter, and the session list in the Claude Code desktop app.

Start on the Code tab

The desktop app has more than one mode. Make sure you are on the Code tab before you follow along. The Cowork tab looks similar but behaves differently.

Where Claude works

Local, and a Working Folder

  • Local vs remoteLeave this on Local. Claude Code can control another machine remotely, but for everyday work you want it acting on your own computer.
  • Working folderThis is the folder Claude can see and act in. Choose a different folder for each kind of work. Lesson 04 explains why one folder per purpose keeps Claude organized.
  • Additional foldersNeed Claude to also read files that live somewhere else, such as your Documents? Add them here so they are in reach without changing your main folder.
Staying in control

Permission Mode and Attachments

  • Permission-mode selectorSets how often Claude checks with you before acting: Ask, Accept edits, or Plan. This is your main safety control. Lesson 05 covers each mode in detail.
  • AttachmentsAdd files to a message here: log files, a spreadsheet, an image. On the desktop app you can also paste an image straight into the input, which the CLI cannot do as easily.
Capability and cost

Model, Effort, and Usage

  • Model selectorChoose which model handles the task. Heavier models think more and cost more. Lesson 03 walks through Haiku, Sonnet, and Opus.
  • EffortA slider for how much reasoning Claude spends before answering. Medium to high is a sensible default.
  • Usage meterShows how much of your rolling and weekly limits you have used. Worth glancing at, especially on heavier models. Lesson 03 explains the limits.
Working in parallel

Sessions and Routines

  • New sessionStart a fresh session for each task with Ctrl/Cmd + N. Lesson 04 explains why one task per session matters.
  • Session listYour running sessions appear together. A session that needs your attention shows a small dot, so you can let work run in the background and check back. Open one to read its transcript or mark it complete.
  • RoutinesFound near the top of the window, routines let Claude run a task on a schedule and notify you. Lesson 06 sets one up.
Key Takeaway

Two controls matter most as you start: the working folder (what Claude can see) and the permission mode (what Claude can do without asking). Get comfortable with those two and the rest of the window falls into place.
