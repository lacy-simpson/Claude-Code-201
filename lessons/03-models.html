Capability and cost

Models, Effort & Usage

Two controls decide how much horsepower Claude brings to a task: the model and the effort level. They also decide how fast you move through your usage. The habit to build is simple: start with the lightest combination that can do the job, and move up only when the work needs it.

Lesson Objective

Choose a model and effort level that fit a task, and read your usage so you know how much headroom you have.

Model and effort

Choosing Models and Effort

The model sets the level of capability Claude brings. Effort controls how much reasoning it spends before answering. Heavier choices can improve hard work, but they use more tokens and move you through your limits faster.

Models change often

Model names, defaults, and available settings change over time. Check the current Claude menu and UiPath guidance before choosing.

Move up on purpose

If the first answer is too shallow, raise the model, the effort, or both. For most work, start with Sonnet at medium effort and reserve Opus for code and heavy building.

Context size

200K vs 1M Context

Some models offer a one-million-token context, five times the default of two hundred thousand. Context is Claude's working memory for the task at hand. The larger window costs extra and uses your limits faster, so the default is the right choice unless you are genuinely working across a very large amount of material. Keeping each task in its own session, which the next lesson covers, usually matters more than a bigger window.

Know your headroom

Reading Your Usage

Claude Code shows your usage in the window. There are a few limits worth understanding so a heavy task does not surprise you.

  • 5-hour limitA rolling limit that starts counting from your first message in a session. How much you get can vary with overall demand, so it may feel tighter during busy hours.
  • Weekly limitA separate allowance that resets every seven days.
  • Extra usageUiPath provides additional headroom beyond those limits. If you hit a limit, you can keep working against that extra usage, and you can request more if you need it.
Don't ration yourself into worse work

The limits exist to be used. Pick the model and effort the task deserves rather than always defaulting to the lightest option to save tokens. If you run out, extra usage is there and more can be requested.
