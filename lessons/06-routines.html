Let work run on a schedule

Routines & Automation

A routine is a task you ask Claude to run on a schedule instead of kicking off by hand each time. It is how a one-off prompt becomes something that quietly keeps you informed without you remembering to ask.

Lesson Objective

Describe what a routine is, where to create one, and recognize a task in your own work that a routine could handle.

What it is

A Task on a Timer

You find routines near the top of the Claude Code window. A routine pairs a prompt with a schedule: every hour, every morning, or whatever cadence fits. When it runs, Claude does the task and can notify you, for example by sending a message in Slack or Teams, so the result comes to you instead of you going to look for it.

A concrete example

A Daily Folder Summary

Say a shared folder collects reports through the day and you want a quick read each morning. Instead of opening it and skimming, you set a routine.

  • Write the task"Each weekday at 8am, summarize any new files in this folder into five bullets, and call out anything that looks urgent."
  • Set the scheduleChoose the cadence in the routine settings, such as every weekday morning.
  • Choose how you hear about itAsk Claude to send the summary to you on Slack or Teams so it lands where you already work.

The same shape works for many jobs: a recurring digest, a periodic check on a status page, or a reminder built from whatever Claude can see.

The same care applies

A routine acts on your behalf while you are not watching, so scope it tightly and prefer read-and-report tasks over ones that change systems. Everything from the permissions lesson still holds when the work runs on a timer.
