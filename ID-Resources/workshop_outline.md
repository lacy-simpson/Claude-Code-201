### Claude Introduction (5 min)

- What Claude is: AI assistant, not a search engine or a chatbot
- What makes it different: context-aware, conversational, generative
- How to think about using it: thinking partner, first-draft generator, research synthesizer
- How not to use it: as a source of truth, for tasks requiring real-time or proprietary data, for decisions that require human judgment without a review step

### The Tools: Chat, Cowork, Code (9 min)

For each tool: one-sentence definition, who it's for, and one concrete use example.

**Claude Chat** (claude.ai / desktop / mobile)
- The core conversational interface. For everyone.
- Use example: drafting a follow-up email, summarizing a document, brainstorming ideas
- Focus of this session

**Claude in Cowork mode**
- Desktop tool for file and task automation. For non-technical users who want to automate file-based workflows.
- Use example: reading a folder of support tickets and generating a summary report

**Claude Code**
- Command-line tool for developers. For engineers who want to delegate coding tasks.
- Use example: reviewing a pull request, writing unit tests, debugging a function

*Sessions 2+ will go deeper on Cowork and Code. Today we focus on Chat as the universal starting point.*

### Set Up Claude for Your Work (9 min)

**Goal:** Everyone confirms they are using the right account and adds just enough durable context to make Claude more useful.

- **Account check (2 min):** Confirm the UiPath Teams or Enterprise account before adding work context.
- **Instructions for Claude (3 min):** Add stable guidance about role, audience, tone, formatting, technical level, editing style, and feedback preferences. Keep it sanitized.
- **Usage awareness (1 min):** Show where usage appears and remind learners to start with default model and effort.
- **Customization orientation (1 min):** Point out Skills, connectors, plugins, and permissions without deep-diving.
- **Projects (2 min):** Show when to use a Project for repeatable context.

---

### Use Claude Chat for a First Task (9 min)

**Goal:** Everyone practices the minimum useful chat pattern: choose the right space, give enough context, steer once, and review before using the answer.

- **Choose the chat space, model, and effort (2 min):** Regular chat, incognito chat, or Project; choose the lightest model and lowest effort that can do the job. Too light can produce shallow results; too heavy can consume more usage than needed.
- **Give enough context (2 min):** Goal, context, output format, and a review cue.
- **Steer with a follow-up (2 min):** Shorten, change format, adjust tone, or add missing detail.
- **Review before using (2 min):** Check facts, assumptions, names, numbers, dates, policy, and customer commitments.
- **Low-risk practice (1 min):** Try one simple work task and one follow-up.

**Facilitator tip:** Keep this section deliberately light. The goal is not prompt mastery; it is helping learners feel the basic loop of start, steer, and review.

---

### Demo (4 min)

**Goal:** One concrete moment where the audience sees Claude do something useful and thinks "I could use that tomorrow."

**What to show:** The Project context payoff. This closes the loop on the personalization segment.

- **Setup (30 sec):** "I've created a Project and given Claude a few sentences about my role. Watch what happens when I ask the same question two ways."
- **Take 1 — no context (1 min):** Open a blank chat. Ask: "Help me write a follow-up email after a customer meeting." Show the output — functional but generic. Point out what Claude had to assume.
- **Take 2 — with Project context (1 min):** Switch to your Project. Ask the exact same thing. Show how Claude uses your role, tone preferences, and audience. The output is noticeably more usable with zero extra prompting.
- **Landing line (30 sec):** "The Project context did the work. You set it up once and every conversation in that Project starts from that baseline."
- **One follow-up prompt (1 min):** Ask Claude to adjust something — shorter, different tone, add a detail. Show that you don't start over; you redirect. Normalize iteration as the workflow.

**Facilitator tip:** Use your actual job context, not a made-up persona. The audience will trust it more and it models exactly what you just asked them to do. Avoid anything that requires explaining background context before the demo starts.

## Facilitator Notes

**Tone:** Practical and grounded. Participants should leave with one thing they can do today, not just inspiration.

**Audience variance:** Expect a wide range of prior exposure. Some will have been using Claude for months; others are on day one. Move through Setup faster for experienced groups.

**Demo prep:** Have your own Project context written and ready. Use a real, sanitized work example — not a generic placeholder.

**Pacing risk:** Setup + Personalization back-to-back is 18 minutes of procedural content. Break it up with brief "here's why this matters" examples between sections to maintain energy.

**Follow-up:** Post the Skill Jar leave-behind resource in the session Slack thread immediately after. Offer a 30-day check-in session where participants share what worked and what questions emerged from real use.
