Keep Claude sharp

Context & Sessions

Claude works inside a context window: its working memory for the current session. When that memory fills up, quality drops in ways that are easy to mistake for the model getting worse. A little discipline about sessions keeps Claude accurate.

Lesson Objective

Explain what compaction is, recognize when it has happened, and use one session per task to avoid it.

What fills up

The Context Window

Everything in a session shares one context window: your messages, Claude's replies, the files it has read, and the system instructions it always carries. As a session grows, that window fills.

When it gets close to full, Claude compacts the conversation. It summarizes what came before into a much shorter form so it can keep going. That summary keeps the gist, but detail is lost. If you were deep in a set of logs or a long draft, some specifics may quietly disappear.

Compaction loses detail

After a compaction, Claude may forget a specific value, name, or instruction from earlier. If an answer suddenly seems vaguer or contradicts something you established, compaction is a likely cause. Re-state the detail it lost and carry on.

The habit that prevents it

One Session per Task

The most common way to fill the window too fast is running several unrelated tasks in one long session. Beyond using up the window, it invites confusion: details from one task bleed into another.

Picture one session where you first draft a release announcement, then switch to analyzing a budget spreadsheet. The two share a window, so there is less room for each, and Claude can start mixing them: a number from the spreadsheet wanders into the announcement, or the tone of the email colors the analysis.

  • New session per taskStart a fresh session with Ctrl/Cmd + N for each distinct task. Each one gets the full window to itself.
  • Close when doneWhen a task is finished, let that session go rather than reusing it for the next, unrelated thing.
  • Reset if it driftsIf Claude starts referencing the wrong task, stop and start a clean session rather than fighting the confusion.
Key Takeaway

One task, one session. It keeps each task in clean working memory, avoids cross-contamination, and delays compaction. It is the cheapest reliability habit you have.
